# Hexo 命令有许多，以下只列出下列常用的 Hexo 命令（以下命令不区分大小写）

# hexo generate

hexo g

生成网站静态文件

  • 便于手动部署网站或查看网站生成的静态文件。
  • hexo g 是 hexo generate 的缩写，两个命令效果一致。

# hexo deploy

hexo d

生成网站静态文件，并部署到所设定的仓库中

  • hexo d 是 hexo deploy 的缩写，两个命令效果一致。

# hexo clean

hexo clean

清除网站缓存文件和已生成的静态文件

  • 网站显示异常可以执行这条命令后再执行 hexo g 命令试试。

# hexo server

hexo s

启动本地服务器，预览网站主题。默认地址为 http://localhost:4000/

  • hexo s 是 hexo server 的缩写，两个命令效果一致。
  • 在本地预览的同时可以修改网站和文章内容，修改完成保存刷新即可。
  • 对 hexo 根目录_config.yml 的修改，需要重启本地服务器后才能预览效果。

# hexo new ""

hexo new "hexo"

新建一篇标题为 “hexo” 的文章

  • 文章标题可以在主题文件夹对应的 md 文件里查找进行修改

# hexo algolia

export HEXO_ALGOLIA_INDEXING_KEY=your apiKey

hexo algolia

新建的文章名称全部添加到 algolia，在站内根据关键字就能搜索到指定文章

  • 在执行 hexo algolia 之前必须先执行 hexo g 生成静态文件，别问为什么 (#｀-_ゝ -)
